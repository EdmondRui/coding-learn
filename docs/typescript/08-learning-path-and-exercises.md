# 第 8 章：学习路线与练习建议

## 8.1 推荐学习顺序

建议你按下面顺序学习：

1. 基本概念和开发环境
2. 基础类型
3. 函数类型
4. 对象类型和类型别名
5. 联合类型和类型收窄
6. 泛型基础
7. 工具类型
8. `keyof`、`typeof`、索引访问类型
9. 判别联合
10. 类、模块、声明文件
11. 条件类型和映射类型

这个顺序的核心原则是：先会建模，再学抽象。

## 8.2 四阶段学习路线

### 第一阶段：基础巩固

目标：

- 会写基本类型
- 会给函数写类型
- 能看懂简单报错

建议练习：

- 写 20 个简单函数
- 给数组练习题加类型
- 用类型描述常见对象

### 第二阶段：建模能力

目标：

- 会写对象结构
- 会用联合类型表达状态
- 会进行基本收窄

建议练习：

- 建模用户系统
- 建模商品和订单
- 建模请求状态

### 第三阶段：复用能力

目标：

- 能写简单泛型
- 会使用工具类型
- 会设计通用返回结构

建议练习：

- 写 `getFirst<T>`
- 写 `ApiResponse<T>`
- 为更新接口设计 `Partial<T>`

### 第四阶段：项目实践

目标：

- 在真实项目中持续使用 TypeScript
- 能读懂复杂一点的报错
- 能避免滥用 `any`

建议练习：

- 用 TypeScript 写一个小型 Node.js 项目
- 给前端接口返回值建模
- 给表单状态和路由参数加类型

## 8.3 每章学习方法

每学一章，建议这样做：

1. 先读概念
2. 亲手敲一遍示例
3. 改错几次看报错
4. 做 3 到 5 个小练习
5. 用一个真实场景复用本章内容

## 8.4 适合练手的小项目

- 待办事项应用
- 用户登录和权限系统
- 商品列表和购物车
- 博客后台数据模型
- 简单记账工具

这些项目足够覆盖：

- 基础类型
- 对象建模
- 接口返回值
- 状态建模
- 泛型复用

## 8.5 学习 TypeScript 时最该培养的能力

不是“背语法”，而是这几种能力：

- 建模能力
- 收窄能力
- 约束能力
- 重构能力

如果你能把真实业务结构准确表达成类型，TypeScript 才真正发挥价值。

## 8.6 一组循序渐进的练习题

### 基础题

1. 定义一个 `Product` 类型，包含 `id`、`title`、`price`
2. 写一个函数，输入两个数字，返回较大值
3. 写一个 `string | number` 参数函数，并正确收窄

### 建模题

1. 设计一个用户角色类型，只允许 `admin`、`editor`、`user`
2. 设计一个订单状态类型，要求状态之间互斥
3. 设计一个接口返回值 `ApiResponse<T>`

### 泛型题

1. 写一个返回数组第一个元素的泛型函数
2. 写一个键值对生成函数
3. 写一个 `Record<string, User>` 结构

### 实战题

1. 为一个商品列表接口建模
2. 为一个登录表单建模
3. 为一个异步加载状态建模

## 8.7 学习中的判断标准

你可以用这些问题检查自己是否真的学会了：

- 我能不能给一个真实函数写出清晰类型？
- 我能不能把一份接口 JSON 建模出来？
- 我能不能理解一个报错为什么出现？
- 我能不能避免用 `any` 糊过去？
- 我能不能用联合类型表达状态，而不是用一堆可选字段凑？

如果这些问题大多都能答出来，说明你已经有比较扎实的 TypeScript 基础。
